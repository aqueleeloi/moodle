O Ollama não está a detetar a gráfica: Por vezes, o Ollama não apanha a configuração do hardware (CUDA/placa gráfica dedicada) e tenta correr o modelo usando apenas o processador principal (CPU). Se isso acontecer, até um PC topo de gama engasga.Vamos tirar a prova dos nove e contornar esse apagão automático, isolando o problema:1.Limpa a VRAM:Antes de começar, fecha qualquer software de design 3D, edição de vídeo ou processos pesados que possam estar a usar a memória da tua placa gráfica em background.2.Testa o modelo diretamente no terminal:Abre o terminal (ou a linha de comandos) e ignora o OpenClaw por agora. Escreve: ollama run qwen2.5-coder:7b. Isto vai obrigar o Ollama a fazer o download de novo e a carregar o modelo de forma 100% isolada, sem regras de timeout externas a atrapalhar.3.Monitoriza o hardware:Se o modelo carregar no terminal, faz-lhe uma pergunta qualquer. Enquanto ele estiver a gerar a resposta, abre o Gestor de Tarefas do Windows (separador Desempenho) ou o monitor do sistema. Verifica se é a tua GPU (Placa Gráfica dedicada) que está a trabalhar. Se vires o teu CPU a 100% e a placa gráfica a dormir, significa que o Ollama não instalou os drivers corretos para usar o teu hardware.
